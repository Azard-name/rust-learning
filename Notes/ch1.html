Installation & Hello World

Rust s'installe via rustup (gestionnaire de versions). Une fois installé, le premier programme :

fn main() {
    println!("Hello, world!");
}

Compilé et exécuté avec rustc main.rs puis .\main.exe — mais en pratique on passe directement par Cargo.

Cargo — Build system & manager de projet

Cargo prend en charge la construction du projet et les librairies dont on a besoin. Un peu comme package.json en JS.

Créer un projet

cd "CHEMIN_DACCES_OU_JE_VEUX_LE_DOSSIER"
cargo new NOM_DU_DOSSIER
cd NOM_DU_DOSSIER

Structure du Cargo.toml

SectionRôle
[package]Config du projet (nom, version, edition). Comme un <header> en HTML.
[dependencies]Liste des crates externes utilisées. Cargo les télécharge automatiquement au build.
 
[package]
name = "hello_cargo"
version = "0.1.0"
edition = "2024"

[dependencies]
# ex : rand = "0.8.5"
📎 Le Top-level project directory contient : README, License, fichiers de config — tout ce qui n'est pas du code. Le code lui-même va dans /src, géré automatiquement par Cargo.

Commandes Cargo essentielles

CommandeCe que ça fait
cargo new NOMCrée un nouveau projet
cargo buildCompile sans exécuter → crée un fichier debug dans target/debug/
cargo runCompile + exécute en une seule commande ✅
cargo checkVérifie les erreurs sans produire d'exécutable — plus rapide, à utiliser pendant le dev
cargo build --releaseCompile avec optimisations pour la release finale
 
cargo build
    Compiling hello_cargo v0.1.0 (file:///projects/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 2.85 secs

Crée un fichier de debug dans le directory, exécutable avec :

.\target\debug\hello_cargo.exe

Mais on peut compiler et exécuter d'un coup avec :

cargo run
     Finished dev [unoptimized + debuginfo] target(s) in 0.0 secs
     Running `target/debug/hello_cargo`
Hello, world!
💡 cargo run = cargo build + .\target\debug\hello_cargo.exe en une commande. C'est celle à retenir. 
cargo check
    Checking hello_cargo v0.1.0 (file:///projects/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 0.32 secs
⚠️ cargo check vérifie sans produire d'exécutable — plus rapide. À utiliser après chaque Ctrl+S pour vérifier que le code compile. (chouette, après le ctrl+s, le cargo check à pas oublier. ça va définitivement bien se passer n'est-ce pas ?)
📎 Récap Cargo :
  • Créer un projet : cargo new
  • Builder : cargo build
  • Builder + exécuter : cargo run
  • Vérifier sans binaire : cargo check
  • Cargo stocke le build dans target/debug, pas dans le dossier du code
  • Les commandes sont identiques peu importe le système (et ça, c'est bien)

Lorsque le projet est prêt pour la release :

cargo build --release
⚠️ --release allonge la compilation mais accélère l'exécution. À faire en dernier, juste avant de distribuer. Finir le projet en entier avant d'optimiser.

Crates — Standard vs Non-standard

Standard (disponible sans rien ajouter au Cargo.toml) : types de base String, Vec, HashMap, Option, Result, I/O, threads, itérateurs, maths de base.

Non-standard (à ajouter dans [dependencies]) :

BesoinCrate
HTTP / requêtes webreqwest
Sérialisation JSONserde
Base de donnéessqlx, diesel
Interface graphiqueegui, iced
Génération aléatoirerand
Dates et heures avancéeschrono
Async / concurrence avancéetokio

Les crates sont sur crates.io — l'équivalent de npm pour Rust. Cargo les télécharge automatiquement au cargo build.
