Snippets

Le Rust est un langage de pré-compilation : tu peux créer et compiler le programme puis le donner à exécuter à quelqu’un d’autre sans impliquer une installation du langage, à l’instar du Python, JavaScript ou Ruby.

En conclusion : c’est plutôt pas mal pour les virus et pour créer des programmes en “environnement stérile” (hors-ligne) à faire basculer en entreprise.

Hello, Cargo !

Le Cargo c’est le système et le manager de projet du Rust. Il prend en charge la construction et les librairies dont on a besoin dans le code (librairie initiale et créer spécifiquement pour le code en cours [dependencies].

Dans le cas spécifique du Hello World qu’on vient de créer, il n’y a pas de dependencies, puisqu’il s’agit de l’exécution de la librairie de base.

—> Il y aura fatalement besoin de rajouter les librairies pour des fonctions non standard.

Construction du fichier TOML :

[package]Section qui contient les configurations du fichier.
Les trois lignes suivantes sont les informations à compiler qui servent de config.
Un peu comme le <header> en HTML visiblement.
[dependencies]Section où JE liste les crates que j’utilise.

Snippets

[package]
name = "hello_cargo"
version = "0.1.0"
edition = "2024"

[dependencies]

Par défaut, Cargo héberge le code dans un dossier src. D’où sa fonction de manager : il range tout tout seul, y’a plus qu’à suivre le blue print.

cargo build 
    Compiling hello_cargo v0.1.0 (file:///projects/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 2.85 secs

Créer un fichier de debug dans le directory, qu’on exécute avec la commande suivante :

.\target\debug\hello_cargo.exe

Mais on peut aussi compiler l’intégralité d’un coup avec : 

cargo run
		 Finished dev [unoptimized + debuginfo] target(s) in 0.0 secs
     Running `target/debug/hello_cargo`
Hello, world!

La dernière commande est plus pratique de se rappeler des deux précédentes pour le même résultat.

cargo check
    Checking hello_cargo v0.1.0 (file:///projects/hello_cargo)
    Finished dev [unoptimized + debuginfo] target(s) in 0.32 secs

Ce code-ci vérifie le code sans produire d’executable comme le ferait le build, ce qui le rend plus rapide. C’est la commande à utiliser pour vérifier pendant qu’on code (logique) et vérifier en permanence que le code compile (chouette, après le ctrl + s, le cargo check à pas oublier. ça va définitivement bien se passer n’est-ce pas ?).

Lorsque le project est prêt pour la release, on utilise : 

cargo build --release

La commande lance la compilation avec optimisation en créant un fichier de release.

A noter que la compilation s’allongera avec la présence de l’exécutable mais fera lire le fichier Rust plus vite. Le plus sage est donc de réaliser le fichier en entier AVANT de l’optimiser et de ne pas oublier cette étape avant la release pour surveiller le temps d’exécution.
